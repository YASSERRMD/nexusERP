# NexusERP

This repository contains a research implementation generated using Chat.z.ai Agent Mode powered by GLM-5.

## Research Context

NexusERP was generated to evaluate GLM-5’s capability in designing complex enterprise system architectures.

The benchmark emphasizes:

- Multi-tenant ERP modeling
- Double-entry accounting integrity
- Inventory valuation logic
- Transaction-safe operations
- Modular domain separation

## Project Overview

NexusERP includes:

- Organization and user management
- Role-based access control
- Chart of Accounts hierarchy
- Double-entry ledger engine
- Sales and procurement workflows
- Inventory management with FIFO valuation
- Payroll calculation module
- Financial reporting logic

## Technology Stack

- Next.js 16 (App Router)
- Node.js API Routes
- Prisma ORM
- SQLite
- In-memory caching
- Socket.io
- Docker

## Research Objective

This project evaluates GLM-5’s ability to:

- Model relational complexity correctly
- Enforce financial balancing rules
- Maintain transactional integrity
- Integrate financial and inventory logic
- Preserve architectural consistency across modules
